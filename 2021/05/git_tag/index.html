Git tag - 打标签操作

2021-05-08
字数: 906 阅读时长: 6mins 阅读次数: 次
分类： 其它
标签： git

Git 可以给仓库里的分支打上标签做标记，比如在代码发布后经常会打上V1.0，V2.0 这样的标签，本文看一下这些打标签相关的操作。

本文中示例 $ 开头的表示执行的命令，其他内容为命令的输出。

创建标签

通过 git tag 即可很容易的创建标签：

为当前提交创建标签

git tag

示例：

使用 git log 也能看到打的标签。

为历史提交创建标签

为历史提交创建标签，则在后面加上 commitHash 即可。

git tag

示例：

标签增加备注

使用 -m 可以为标签增加附属信息：

git tag -m ““

可以看到输出里多了打标签人、打标签的日期时间、记录的附注信息。

查看标签

查看当前代码库全部的标签

git tag

注意，标签是按字母排序而非提交日期或者标签创建日期排序。

使用通配符查看指定标签

如果代码仓库有很多标签，也可以使用通配符匹配符合指定格式的标签：

git tag -l ““

查看某个标签的详细信息

git show

标签推送到远程仓库

推送指定标签到远程仓库

git push origin

推送全部标签到远程仓库

可以把本地存在而远程仓库不存在的标签一次性推送到远程仓库。

git push origin –tags

删除标签

删除本地标签

git tag -d

删除远程仓库的标签

git push :refs/tags/
or
git push origin –delete

代码切换到指定标签

使用 git checkout 命令，可以使代码回到指定的标签位置，但此时会处于 detached HEAD 状态：

git checkout

当然也可以在 checkout 的时候创建一个新分支：

git checkout -b
